What happens when you initialize a repository? Why do you need to do it?

	When you initialize a repository, git creates an empty repository with no commits in it. 
	This step is necessary because many Git commands do not work outside of an initialized repository.

What value do you think it offers?
	
	The staging area is an intermediate area where you can add changes before committing them to the repository. This allows add changes to the staging area by-one-by, and committing only when a logical change is been made, making the commit history more readable.
